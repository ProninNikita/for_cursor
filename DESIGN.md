# Squad Tactics — Дизайн-документ

## Концепция
Пошаговая стратегия с системой составления команды. Персонажи получаются разными способами, включая лутбоксы. При выпадении из лутбокса генерируется **полностью новый** персонаж с уникальной предысторией, характером, классом и способностями.

---

## Цепочка генерации персонажа (лутбокс)

```
Случайная предыстория → Черты характера → Класс → Базовые способности → Уникальные способности
```

### 1. Предыстория (Backstory)
- Происхождение: крестьянин, дворянин, солдат, учёный, преступник, странник...
- Ключевое событие: потеря семьи, обучение у мастера, предательство, открытие таланта...
- Мотивация: месть, защита, знание, богатство, искупление...

### 2. Характер (Personality)
Черты характера влияют на:
- **Класс**: агрессивный → воин, расчётливый → тактик, мистический → маг
- **Базовые статы**: смелость → сила, осторожность → защита
- **Стиль способностей**: импульсивный → мощные, но с откатом; терпеливый → накопительные

### 3. Класс
Определяется предысторией + характером:
- Воин, Маг, Лекарь, Разведчик, Защитник, Берсерк, Тактик, Ассасин...

### 4. Способности
- **Базовые**: из пула класса (2–3 на класс)
- **Уникальные**: генерируются из комбинации предыстория + характер + класс
- Пример: "Бывший палач + месть + воин" → способность "Казнь" (доп. урон цели с низким HP)

## Системы

### Боевая система (пошаговая)
- Сетка/гексы или карточная система
- Очерёдность ходов (инициатива)
- Действия: перемещение, атака, способность, защита, предмет
- AP/очки действия за ход

### Составление команды
- Ростер: все полученные персонажи
- Отряд: до N персонажей на миссию (например, 4)
- Синергии между персонажами (опционально)

### Способы получения персонажей
1. **Лутбоксы** — полная процедурная генерация
2. **Сюжет** — фиксированные персонажи
3. **Магазин/награды** — (если понадобится)

---

## Структура данных персонажа

```gdscript
Character:
  - id: String (уникальный)
  - name: String (генерируется или из пула имён)
  - backstory: BackstoryData
  - personality: PersonalityData  
  - character_class: String
  - stats: { hp, atk, def, speed, ... }
  - abilities: Array[Ability]
  - portrait_path: String (опционально)
```
