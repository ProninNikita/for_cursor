=== SQUAD TACTICS — ПЛАН РАЗРАБОТКИ ===

--- УЖЕ СДЕЛАНО ---
• Генерация персонажей на основе предыстории и характера с классами
• Лутбоксы — процедурная генерация новых персонажей
• Ростер персонажей

--- ИДЕИ ИЗ ОБСУЖДЕНИЯ ---

1. ОБУЧЕНИЕ НА ОСНОВЕ ДЕЙСТВИЙ
   • Не только атаки — практически любое действие (лечение, баф, блок, перемещение и т.д.)
   • Персонаж взял нож и бьёт им → качает силу → получает навык, связанный с ножами
   • Решает проблему "5 лекарей выпало" — лекарь может развиться в бойца через действия

2. МУЛЬТИКЛАСС С ОГРАНИЧЕНИЯМИ
   • Лекарь может взять нож и начать бить — учится через действия
   • Мечник НЕ знает лечение — получает способности иным способом:
     - Чтение книг на базе
     - Обучение у другого персонажа (учитель должен владеть навыком)
   • Синергия: мастер-лекарь в отряде = возможность прокачать лечение другим

3. ЛИМИТ УНИВЕРСАЛА
   • В теории можно стать универсалом, но:
   • Чем больше специализаций — тем слабее можно прокачать каждую
   • 1 специализация → до Мастера
   • 2-3 специализации → макс. Профи
   • 4+ → макс. Умелец

4. СИСТЕМА МАСТЕРСТВА (не как в других играх)
   • Диапазоны: Новичок → Умелец → Профи → Мастер
   • Внутри диапазона: поднимаешь статы до потолка
   • Выход на следующий диапазон: по удаче/прорыву
   • Очень маленький шанс при значимых действиях:
     - Убийство врага
     - Оживление союзника
     - И т.п. — момент "осознания границ"
   • Вопросы: шанс на действие или на бой? Разный вес для босса/слабого врага?
   • Шанс растёт с опытом в диапазоне или фиксирован?

5. ПРЕДЫСТОРИЯ
   • Влияет на стартовые способности
   • Предрасположенности: больший шанс прорыва в "своей" ветке
   • Влияние на скорость обучения (книги, учителя) — учёный быстрее по книгам

6. ЗАБВЕНИЕ
   • Пока не решено
   • Варианты: без забвения / пассивное / только при смене специализации

7. ИВЕНТЫ (из исходного плана)
   • Получение способностей через ивенты (состояние при смерти → берсерк и т.д.)

--- ГОРОД-ХАБ ---

Центральная локация, откуда игрок управляет всеми системами:

1. ПОРТАЛ
   • Призыв героев (лутбоксы)
   • Входная точка для новых персонажей

2. БАШНЯ
   • Отправка героев в бой
   • Выбор миссий / походов
   • Связь с боевой системой

3. МАСТЕРСКАЯ
   • Создание предметов для героев
   • Крафт экипировки, расходников

4. ОСОБНЯК (Дом)
   • Герои отдыхают
   • Взаимодействие с героями (диалоги, отношения)
   • Возможно: книги для обучения (из плана мультикласса)

5. ТРЕНИРОВОЧНАЯ ПЛОЩАДКА
   • Тренировка навыков и статов
   • Прокачка без боя (из плана обучения на основе действий)

--- ЖИВОЙ ГОРОД (дни, задания, еда) ---

• Время идёт по дням
• Жители = персонажи из ростра
• Каждому жителю можно дать ОДНО задание в день
• Типы заданий: экспедиция, крафт, тренировка, отдых и т.д.
• Еда — ресурс выживания: каждый персонаж съедает N еды в день
• Пока есть еда — можно управлять городом, собирать ресурсы
• Пополнение еды: экспедиции, покорение башни (миссии)
• Цикл: день → потребление еды → задания → результаты → следующий день

--- СТРАТЕГИЯ РАЗРАБОТКИ (MVP) ---

Приоритет: сначала играбельный цикл, потом глубина.

MVP (минимальный играбельный цикл):
1. Базовый бой — пошаговый, простой (атака, способность, защита)
2. Башня — 1-2 этажа, даёт еду
3. Дневной цикл — 1 еда на персонажа, кнопка "Завершить день"
4. Задания — 2-3 типа: башня, отдых, тренировка (упрощённая)

Цель: замкнуть петлю еда → задания → башня → еда

Порядок добавления после MVP:
1. Прокачка через действия
2. Мастерство (диапазоны, прорывы)
3. Мультикласс (книги, обучение)
4. Экспедиции
5. Крафт
6. Отношения

Упрощения на старте:
• Прорывы — сначала детерминировано (10 боёв = прорыв), RNG позже
• Задания — начать с 3-4: башня, отдых, тренировка, охота
• Мультикласс — сначала только через действия, книги/обучение позже
• Забвение — не делать, усложняет без явной пользы
